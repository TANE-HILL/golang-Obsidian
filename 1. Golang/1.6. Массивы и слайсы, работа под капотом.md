# Массив
Это последовательность [[1.1. Переменные|элементов]] [[1.2. Типы данных|одного типа]] фиксированной длины(коробка "фиксированного размера", в которой хранятся элементы одного типа.).

![[Pasted image 20260717142148.png|422]]

В [[1.0. Go Basics|Go]] массивы имеют **жёсткий размер**, который задаётся при создании.
	`[3]int` — это всегда ровно 3 целых числа.

#### Примеры:
```go
var arr1 [3]int        // массив из 3 целых чисел
var arr2 [5]string     // массив из 5 строк
var arr3 [2]bool       // массив из 2 булевых значений
```

#### Если мы не заполняем массив сразу, то все его [[1.1. Переменные|элементы]] получают **нулевое значение**:
- `int` → `0`
- `string` → `""` (пустая строка)
- `bool` → `false`

**Нулевые значения при создании массива:**
```go
var arr [3]int
fmt.Println(arr)
```
**Вывод:**
```Run
[0 0 0]
```

# Особенности массивов:
- **<u>Индексация начинается с 0.</u>**
- Если элемент не инициализировать, он будет иметь **нулевое значение** (`0`, `false`, `""` - в зависимости от типа).
- Размер массива — часть его типа (<u><font color="#f79646">[3]int</font> и <font color="#4bacc6">[4]int</font> — разные типы</u>).
- Попытка <u>обратиться за пределы</u> → `panic: index out of range`.

<u>Инициализация</u> — это когда ты **присваиваешь [[1.1. Переменные|значение переменной]]** (или элементу массива) **при создании**.

Если этого не сделать, [[1.0. Go Basics|Go]] автоматически заполняет такие элементы **значением по умолчанию** для их [[1.2. Типы данных|типа]].

# Инициализация массива

**Можно задать значения сразу:**
```go
arr := [4]string{"Go", "is", "fast", "!"}
fmt.Println(arr)
```
**Вывод:**
```Run
[Go is fast !]
```

**Если использовать `...`, компилятор сам определит размер:**
```go
nums := [...]int{1, 2, 3, 4, 5}
fmt.Println(nums)
```
**Вывод:**
```Run
[1 2 3 4 5]
```

# Работа с элементами

```go
nums := [5]int{10, 20, 30, 40, 50}

fmt.Println(nums[0])   // первый элемент
fmt.Println(nums[4])   // последний элемент
fmt.Println(len(nums)) // длина массива
```
**Вывод:**
```Run
10
50
5
```

# Передача массива в функцию

**При передаче массива в функцию <u>копируется весь массив</u>, а не ссылка:**
```go
func change(arr [3]int) {
    arr[0] = 99
    fmt.Println("Внутри:", arr)
}

func main() {
    nums := [3]int{1, 2, 3}
    change(nums)
    fmt.Println("Снаружи:", nums)
}
```
**Вывод:**
```Run
Внутри: [99 2 3]
Снаружи: [1 2 3]
```

Поэтому массивы почти не используют для работы с данными, чаще применяют **слайсы**.

# Cлайсы

**<u>Слайс</u>** — это **динамическая обёртка над массивом**. Он состоит из:
1. [[1.4. Указатели - работа с памятью и передача данных|Указателя]] на массив.
2. Длины (`len`) — сколько элементов видно.
3. Ёмкости (`cap`) — сколько элементов можно вместить, не создавая новый массив.

**Слайсы** позволяют работать гибко: добавлять элементы, создавать подмножества, копировать.
#### Пример:
```go
nums := []int{1, 2, 3}
fmt.Println(nums, "len:", len(nums), "cap:", cap(nums))
```
**Вывод:**
```Run
[1 2 3] len: 3 cap: 3
```

# Добавление элементов ~ <font color="#ff0000">append</font> ~

**Слайсы можно расширять через `append`:**
```go
nums := []int{1, 2, 3}
nums = append(nums, 4)
nums = append(nums, 5, 6)
fmt.Println(nums)
```
**Вывод:**
```Run
[1 2 3 4 5 6]
```

<u>Под капотом</u>: если места хватает (`len < cap`) — **новые элементы добавляются в тот же массив**. Если места нет — создаётся новый массив в 2 раза больше, данные копируются, и <font color="#ff0000">append</font> возвращает новый слайс.

Старый массив позже будет удалён сборщиком мусора.

# Cоздание слайса через ~ <font color="#ff0000">make</font> ~

Кроме срезов на основе массива и литералов, можно создавать слайсы с помощью функции **`make`**:
```go
nums := make([]int, 3)         // длина = 3, емкость = 3
nums2 := make([]int, 2, 5)     // длина = 2, емкость = 5

fmt.Println(nums, len(nums), cap(nums))
fmt.Println(nums2, len(nums2), cap(nums2))

```
**Вывод:**
```Run
[0 0 0] 3 3
[0 0] 2 5

```

**Важно:**
- `make` используется **только** для срезов, карт и каналов.
- Все элементы инициализируются **нулевыми значениями**.
- Можно управлять `len` и `cap` с самого начала

# Пошаговая визуализация ~ <font color="#ff0000">append</font> ~

*Очень важно понять, как несколько слайсов могут ссылаться на один и тот же массив.*
### Шаг 1. Создаём <u><font color="#4bacc6">x</font></u>
```go
x := make([]int, 3, 6)
x[0], x[1], x[2] = 1, 2, 3
```

```
Базовый массив (cap=6):
[ 1 | 2 | 3 | 0 | 0 | 0 ]
^^^^^^^^^^^^^
  x (len=3, cap=6)
```
### Шаг 2. <u><font color="#4bacc6">y</font> :<font color="#ff0000">=</font> append(<font color="#4bacc6">x</font>, <font color="#8064a2">4</font>)</u>
```go
y := append(x, 4)
```
- `append` добавил `4` в массив на позицию `3`.
- Теперь у `y` длина 4.

```
Базовый массив (cap=6):
[ 1 | 2 | 3 | 4 | 0 | 0 ]
^^^^^^^^^^^^^^^^^
 
  x (len=3)
  y (len=4, cap=6)
```
### Шаг 3. <u><font color="#4bacc6">z</font> :<font color="#ff0000">=</font> append(<font color="#4bacc6">x</font>, <font color="#8064a2">5</font>)</u>
```go
z := append(x, 5)
```
- Мы снова взяли `x` (len=3).
- Go думает: "Места в cap хватает, значит пишу на позицию 3".
- Но там уже стоит `4` (который добавил `y`)!
- Новый элемент затирает старый.

```
Базовый массив (cap=6):
[ 1 | 2 | 3 | 5 | 0 | 0 ]
^^^^^^^^^^^^^^^^^
  
  x (len=3)
  y (len=4)   // теперь видит [1 2 3 5]
  z (len=4)
```
### ИТОГ
```go
fmt.Println(x) // [1 2 3]
fmt.Println(y) // [1 2 3 5]
fmt.Println(z) // [1 2 3 5]
```
**Вывод:**
```Run
[1 2 3]
[1 2 3 5]
[1 2 3 5]
```

# Когда создаётся новый массив

Если `append` переполняет `cap`, Go выделяет новый массив большего размера и возвращает ссылку на него.

```go
a := []int{1, 2, 3}
b := append(a, 4, 5, 6, 7) // cap не хватило
a[0] = 99

fmt.Println(a) // [99 2 3]
fmt.Println(b) // [1 2 3 4 5 6 7]

```

Здесь `a` и `b` уже независимы.

# Срезы массива

**Слайс можно получить из массива или другого слайса:**
```go
arr := [5]int{10, 20, 30, 40, 50}
slice := arr[1:4]
fmt.Println(slice)
```
**Вывод:**
```Run
[20 30 40]
```

Изменения в слайсе отражаются в массиве:
```go
slice[0] = 99
fmt.Println(arr) // [10 99 30 40 50]
```

# Копирование слайсов

**Чтобы скопировать данные в новый массив:**
```go
src := []int{1, 2, 3}
dst := make([]int, len(src))
copy(dst, src)

dst[0] = 99
fmt.Println("src:", src)
fmt.Println("dst:", dst)

```
**Вывод:**
```Run
src: [1 2 3]
dst: [99 2 3]
```

# Нулевой слайс

```go
var s []int
fmt.Println(s, len(s), cap(s)) // [] 0 0

s = append(s, 10)
fmt.Println(s) // [10]

```
`nil`-слайс ведёт себя как пустой список, его можно использовать без инициализации.

# Оператор ~ <font color="#ff0000">range</font> ~

Оператор `range` используется для **итерации** (перебора) элементов в массивах, слайсах, мапах и других коллекциях.

Он возвращает **индекс** и **значение** каждого элемента.

Пример с массивом:
```go
arr := [4]int{10, 20, 30, 40}

for i, v := range arr {
    fmt.Println(i, v)
}
```
**Вывод:**
```Run
0 10
1 20
2 30
3 40
```

Если индекс не нужен — его можно опустить через `_`:
```go
for _, v := range arr {
    fmt.Println(v)
}
```

**Важно:**
- `range` всегда возвращает **копию значения**, а не сам элемент.
    Чтобы изменить элемент — обращайтесь по индексу (`arr[i] = ...`).
- В слайсах и массивах `range` <u>перебирает элементы по порядку.</u>

Пример, как создать **пустой массив из 4 элементов** и **заполнить его через `range`**:
```go
package main
import "fmt"
func main() {
    var arr [4]int // все элементы по умолчанию равны 0

    // Заполним массив числами от 1 до 4
    for i := range arr {
        arr[i] = i + 1
    }

    fmt.Println(arr)
}
```
**Вывод:**
```
[1 2 3 4]
```

**Как это работает:**
- `for i := range arr` возвращает **индексы** от `0` до `3`;
- внутри цикла мы присваиваем каждому элементу `arr[i]` значение `i + 1`.

Если бы ты использовал слайс `(make([]int, 4))`, принцип работы с range был бы точно таким же.

# Копирование элементов срезов ~ <font color="#ff0000">copy</font> ~

`copy(dst, src)` копирует элементы между срезами.

Копия независима от исходного массива:
```go
src := []int{1,2,3}
dst := make([]int, len(src))
copy(dst, src)
dst[0] = 100
fmt.Println(src, dst)
```
**Вывод:**
```Run
[1 2 3] [100 2 3]
```
