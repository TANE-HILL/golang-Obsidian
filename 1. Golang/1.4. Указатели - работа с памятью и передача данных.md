# Зачем нужны указатели?

В [[1.0. Go Basics|Go]] по умолчанию [[1.1. Переменные|переменные]] передаются **по значению** — создаётся копия.

- иногда это удобно (безопасно, мы не портим оригинал), но иногда нужно **работать с оригиналом**;
- чтобы это сделать, нужны **указатели** — они позволяют «работать не с копией, а с самим объектом в памяти».

Указатели — это одна из базовых тем, без которой сложно понимать, как устроен [[1.0. Go Basics|Go]].

**Это особенно важно при работе с функциями, структурами и большими объектами данных.**

# 1. [[1.1. Переменные|Переменные]] и память

Когда мы пишем:
```go
x := 42
```

В памяти создаётся ячейка, где хранится число `42`. У этой ячейки есть **адрес** — уникальный номер в памяти (примерно как квартира в доме).

📌 Аналогия:
- Переменная `x` — это «квартира с мебелью».
- Адрес `&x` — это «номер квартиры».
- Указатель `p := &x` — это «бумажка с адресом квартиры».
- Разыменование `*p` — это «зайти в квартиру и посмотреть на мебель».

# 2. Получение адреса ~ <font color="#ff0000">&</font> ~

**Чтобы узнать адрес [[1.1. Переменные|переменной]], используется оператор <font color="#ff0000">&</font>:**
```go
x := 42
fmt.Println("Значение x:", x)
fmt.Println("Адрес x:", &x)
```
**Вывод:**
```
Значение x: 42
Адрес x: 0xc0000140a8
```

**Адрес** - это **уникальный идентификатор места в памяти**.

# 3. Указатели и разыменование ~ <font color="#ff0000">( )</font> ~

**Указатель хранит адрес переменной.**
```go
x := 10
p := &x // p — указатель на int
```
**Если `p` — это «бумажка с адресом», то `*p` — это «что лежит по этому адресу»:**
```go
fmt.Println("Адрес:", p)
fmt.Println("Значение по адресу:", *p)
```
**Вывод:**
```
Адрес: 0xc0000140a8
Значение по адресу: 10
```

**Мы можем изменить значение через указатель:**
```go
*p = 99
fmt.Println("x:", x) // x тоже изменился
```
**Вывод:**
```
x: 99
```

# 4. **Передача данных:** *копия* или *ссылка*?

По умолчанию [[1.0. Go Basics|Go]] передаёт всё **по значению**.

Это значит: если присвоить одну [[1.1. Переменные|переменную]] другой, создаётся копия:
```go
a := 5
b := a
b = 10
fmt.Println("a:", a, "b:", b)
```
**Вывод:**
```
a: 5 b: 10
```
**Оригинал (`a`) не изменился.**

Но если мы работаем с указателем, то у нас два «ключа от одной квартиры»:
```go
a := 5
p := &a
q := p

*q = 20
fmt.Println("a:", a)
```
**Вывод:**
```
a: 20
```
**Теперь `p` и `q` указывают на один и тот же объект в памяти.**

# 5. Нулевой указатель (`nil`)

**Указатель может быть «пустым» — тогда он равен `nil`:**
```go
var p *int
fmt.Println(p == nil) // true
```

Такой указатель **не указывает ни на что**.

Если попытаться разыменовать `*p`, программа упадёт с ошибкой (panic).

**Поэтому нужно проверять:**
```go
if p != nil {
    fmt.Println(*p)
} else {
    fmt.Println("Указатель пустой")
}
```

# 6. Создание указателя через ~ <font color="#ff0000">new</font> ~

Можно создать «**место в памяти**» сразу, **без переменной**:
```go
p := new(int) // выделили память под int, получили *int
fmt.Println(*p) // 0 — нулевое значение
*p = 42
fmt.Println(*p) // 42
```
**`new(T)` создаёт указатель на «пустое значение типа T».**

📌 *Например*:
- `new(int)` → `int`, внутри 0
- `new(bool)` → `bool`, внутри false
- `new(string)` → `string`, внутри ""

# 7. Адрес можно брать не у всего

**Можно:**
```go
x := 10
p := &x // ок
```
**Нельзя:**
```go
p := &(10) // ошибка
```
У числа 10 нет своего адреса, потому что он **временный**.

# 8. Работа с памятью и стоимость копирования

#### ⚠️ **Важно понимать:**
- Когда мы копируем **маленькие типы** (`int`, `bool`), это **дёшево и быстро**.
- Когда мы копируем **большие структуры** или **большие массивы**, это может быть **дорого** (нужно продублировать всю память).

Поэтому часто выгоднее хранить и передавать **указатель на объект**, а не **копию объекта**.

Пример:
```go
// Допустим, у нас структура 1 МБ (представим).
// Если передавать её по значению — копируется весь мегабайт.
// Если передать указатель — копируется только адрес (8 байт).
```
📌 Это не значит, что всегда нужно использовать указатели. **Иногда копия проще и безопаснее**. Главное — понимать **баланс**.

# 9. Чаще всего указатели нужны в [[1.5. Функции и передача данных|функциях]]

Обычно указатели используются, чтобы **[[1.5. Функции и передача данных|функция]] могла менять аргументы**:
```go
func addOne(n *int) {
    *n = *n + 1
}
```

```go
x := 10
addOne(&x)
fmt.Println(x) // 11

```
