# Зачем нужен map?

Map (или «хэш-таблица») в [[1.0. Go Basics|Go]] — это структура данных, где каждому **ключу** соответствует одно **значение**.

- Например, ключ "Петя", значение — телефон "89112223344".
- Можно мгновенно найти значение по ключу, не перебирая весь список.
# Создание map

#### Способ 1: через `make`
```go
students := make(map[string]int)
```
- `make` — специальная функция для создания динамических структур (map, slice, channel).
- В скобках мы указываем тип:
    - `string` — тип ключа (имя ученика).
    - `int` — тип значения (оценка).
#### Способ 2: через литерал
```go
grades := map[string]int{
    "Аня": 5,
    "Петя": 4,
    "Маша": 3,
}
```
Здесь мы сразу создаём map с готовыми данными.
Ключи — строки с именами, значения — оценки.

# Основные операции с map

#### ==1. Добавление элементов:==
```go
students := make(map[string]int)
students["Иван"] = 5
students["Мария"] = 4

```
- `students["Иван"] = 5` — добавили ключ `"Иван"` и присвоили значение `5`.
- Теперь map хранит 2 пары: `"Иван" → 5`, `"Мария" → 4`.

#### ==2. Получение значения по ключу:==
```go
fmt.Println(students["Иван"])
```
- Мы пишем `students["Иван"]`.
- Go ищет `"Иван"` и возвращает `5`.

**Вывод:**
```Run
5
```
#### ==3. Проверка наличия ключа:==
Иногда ключа может не быть. Тогда возвращается значение по умолчанию:
Для `int` это `0`, для `string` — пустая строка, для `bool` — `false`.

```go
fmt.Println(students["Саша"])
```
**Вывод:**
```
0
```
Но это может запутать: непонятно, «Саша» реально получил 0 или его просто нет.
#### [!!!] **Поэтому в [[1.0. Go Basics|Go]] <u>есть способ проверки через `ok`</u>:**
```go
value, ok := students["Саша"]
if ok {
    fmt.Println("Оценка:", value)
} else {
    fmt.Println("Такого ученика нет")
}
```
#### ==4. Изменение значения==
Map позволяет менять данные по ключу:
```go
students["Иван"] = 3
```
Теперь у Ивана вместо 5 — 3.
#### ==5. Удаление элемента==
```go
delete(students, "Мария")
```
Функция `delete` удаляет запись с указанным ключом.
Если такого ключа нет — ничего страшного не произойдёт.

# Перебор элементов

Map можно обойти с помощью `for ... range` .

**Если в слайсах параметрами были счетчик элемента и значение, то в <font color="#ff0000">Map</font> при переборе через <font color="#ff0000">range</font> параметрами будут <font color="#ff0000">key</font> и <font color="#ff0000">value</font>:**
```go
for name, grade := range students {
    fmt.Println(name, grade)
}
```
**Пример вывода:**
```Run
Иван 5
Мария 4
```

⚠️ Важно: порядок перебора случайный, [[1.0. Go Basics|Go]] не гарантирует порядок ключей: то есть <u>при каждом запуске элементы могут выводиться в разном порядке.</u>

# Особенности ~ <font color="#ff0000">map</font> ~

1. **<u>Ключи уникальны.</u>**
    Если записать один и тот же ключ ещё раз, старое значение заменится новым.

2. **<u>Map хранится в памяти как хэш-таблица.</u>**
    Это значит, что поиск по ключу очень быстрый — почти мгновенный, в отличие от перебора массива.

3. **<u>Ключи должны быть сравнимыми.</u>**
    В Go ключами могут быть: числа, строки, булевы значения, массивы, структуры (если каждое поле структуры - **сравнимый тип**). Про структуры погооврим попхже

    Слайсы, map и функции не могут быть ключами, потому что они несравнимы.

4. **<u>Map — это ссылка.</u>**
    Когда мы передаём map в функцию, передаётся ссылка, а не копия.

    То есть изменения внутри функции будут видны снаружи(не нужно передавать указатель в функцию).

# **Пример реальной задачи**

<u>Задача</u>: посчитать, сколько раз встречается каждое слово в предложении.

```go
text := "go is fun and go is simple"
words := strings.Split(text, " ")

counts := make(map[string]int)

for _, word := range words {
    counts[word]++
}

fmt.Println(counts)
```

1. `strings.Split` разбивает строку по пробелу.
2. Создаём map `counts`, где ключ — слово, а значение — количество.
3. Для каждого слова увеличиваем счётчик `counts[word]++`.

**Вывод:**
```Run
map[and:1 fun:1 go:2 is:2 simple:1]
```

Слово `go` встречается 2 раза, `is` — 2 раза, остальные — по одному.

# Вложенные структуры

map может хранить другие <font color="#ff0000">map</font>, [[1.6. Массивы и слайсы, работа под капотом|слайсы]] и даже [[1.4. Указатели - работа с памятью и передача данных|указатели]] на структуры.

Это полезно для сложных моделей, например: студент → предмет → оценка.

```go
grades := map[string]map[string]int{
    "Vasya": {"math": 5, "phys": 4},
}
fmt.Println(grades["Vasya"]["phys"])
```
**Вывод:**
```Run
4
```

Важно:
- Перед записью во вложенную карту нужно убедиться, что она инициализирована.
- Вложенные карты удобно использовать для группировок.
