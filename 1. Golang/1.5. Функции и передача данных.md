# Зачем нужны функции?

Когда программа становится большой, код начинает повторяться.

**Например, вводится приветствие для разных пользователей:**
```go
fmt.Println("Привет, Аня")
fmt.Println("Привет, Иван")
fmt.Println("Привет, Олег")
```
Если нужно **поменять фразу** (например, «*Добро пожаловать*» вместо «*Привет*»), придётся исправлять во многих местах. Это неудобно и приводит к ошибкам.

👉 Решение: вынести повторяющийся код в **функцию**. Тогда править нужно будет только её.

📌 Функции помогают:
- **Избежать дублирования** (DRY — don’t repeat yourself).
- **Сделать код понятнее** — у функции есть имя, которое говорит «что она делает».
- **Упростить поддержку** — меняем одно место, и поведение меняется везде.
- **Структурировать программу** — разбиваем задачу на маленькие блоки.

Функция — это как «мини-программа» внутри программы: у неё есть вход (**параметры**) и выход (**результат или просто действие**).

# Функции <font color="#ff0000">без возвращаемого значения</font>

Такие функции делают что-то, но ничего не возвращают
#### Пример:
```go
func sayHello() {
    fmt.Println("Привет, мир!")
}

func main() {
    sayHello()
    sayHello()
}
```
**Вывод:**
```Run
Привет, мир!
Привет, мир!
```
Здесь `sayHello` — просто команда: «*напечатай приветствие*». Она ничего не возвращает.

# Функции <font color="#ff0000">с возвращаемым значением</font>

Функции могут возвращать результат с помощью ключевого слова `return`.
#### Пример:
```go
func square(x int) int {
    return x * x
}

func main() {
    result := square(5)
    fmt.Println("Квадрат числа:", result)
}
```
**Вывод:**
```Run
Квадрат числа: 25
```
Здесь функция `square` **получает число и возвращает его квадрат**.
Мы сохранили результат в переменную `result` и напечатали его.
# Основные различия

- **Функции без `return`** — делают действие (печать, запись в файл, отправка запроса).
- **Функции с `return`** — вычисляют и возвращают результат.

📌 Думаем так:
- «*Хочу что-то **сделать***» → функция без `return`.
- «*Хочу что-то **вычислить***» → функция с `return`.

# Параметры функций

Функции могут принимать входные данные (**параметры**).
#### Один параметр:
```go
func greet(name string) {
    fmt.Println("Привет,", name)
}

func main() {
    greet("Аня")
    greet("Иван")
}
```
**Вывод:**
```Run
Привет, Аня
Привет, Иван
```
#### Несколько параметров:
```go
func add(a int, b int) int {
    return a + b
}

func main() {
    fmt.Println(add(3, 4))
}
```
**Вывод:**
```Run
7
```

#### Сокращение:
Если несколько параметров одного типа, можно писать так:
```go
func multiply(a, b int) int {
    return a * b
}
```
#### Область видимости параметров
Каждый параметр функции «живёт» **только внутри** этой функции.
Снаружи — в `main()` или в другой функции — он **не существует**.

Это называется **область видимости**.

**Пример:**
```go
func showNumber(n int) {
    fmt.Println("Внутри функции:", n)
}

func main() {
    showNumber(5)
    fmt.Println(n) // ❌ ошибка: n не видно за пределами функции
}
```
**Функция** — как "<font color="#ff0000">отдельный мир</font>”: всё, что объявлено внутри неё, существует **только в момент её выполнения**.

# Несколько возвращаемых значений

В [[1.0. Go Basics|Go]] функция может вернуть **сразу несколько результатов**.
Это очень удобно.
#### Пример:
```go
func divide(a, b int) (int, int) {
    quotient := a / b
    remainder := a % b
    return quotient, remainder
}

func main() {
    q, r := divide(10, 3)
    fmt.Println("Частное:", q, "Остаток:", r)
}
```
**Вывод:**
```Run
Частное: 3 Остаток: 1
```
#### Игнорирование результата
Если результат не нужен, его можно отбросить через `_`:
```go
q, _ := divide(10, 3)
fmt.Println(q)
```

# Именованные результаты и «голый <font color="#ff0000">return</font>»

#### Что такое сигнатура функции?
**Сигнатура** — это «*паспорт*» функции, в котором указаны **имя**, **параметры** и **возвращаемые типы**.

**Например:**
```go
func stats(a, b int) (sum int, product int)
```
**Здесь:**
- `stats` — имя функции,
- `(a, b int)` — параметры,
- `(sum int, product int)` — **именованные возвращаемые значения**.

В сигнатуре можно сразу назвать возвращаемые значения. Тогда они работают как переменные внутри функции.
#### Пример:
```go
func stats(a, b int) (sum int, product int) {
    sum = a + b
    product = a * b
    return // "голый return" возвращает sum и product
}

func main() {
    s, p := stats(3, 4)
    fmt.Println(s, p)
}
```
**Вывод:**
```Run
7 12
```
⚠️ В длинных функциях лучше писать **явно**, без голого `return`:
`return sum, product`.

# Передача данных в функции: копия или оригинал

Очень важно понимать: в [[1.0. Go Basics|Go]] все параметры передаются **по значению**.

Это значит: функция получает **копию** [[1.1. Переменные|переменной]], **не оригинал**.
#### Пример копии:
```go
func increment(n int) {
    n = n + 1
    fmt.Println("Внутри bump:", n)
}

func main() {
    x := 10
    increment(x)
    fmt.Println("Снаружи:", x)
}
```
**Вывод:**
```lua
Внутри increment: 11
Снаружи: 10
```
**Здесь оригинал `x` не изменился.**
#### ⚠️ Если нужно изменить **оригинал**, используют [[1.4. Указатели - работа с памятью и передача данных|указатели]].

[[1.4. Указатели - работа с памятью и передача данных|Указатель]] — это [[1.1. Переменные|переменная]], которая хранит **адрес другой переменной**.
Если передать в функцию **указатель**, то функция сможет менять значение оригинала.
#### Пример работы с указателем:
```go
func incrementPtr(n *int) {
    *n = *n + 1
}

func main() {
    x := 10
    incrementPtr(&x)
    fmt.Println("После функции:", x)
}
```
**Вывод:**
```Run
После функции: 11
```
- `n` — обращение к значению по адресу;
- `&x` — передаём **адрес** переменной `x`;
- теперь функция работает с самим объектом, а не с копией.

# Вариативные параметры

Иногда нужно принимать **любое количество аргументов**. Для этого пишем `...тип`.
#### Пример:
```go
func sum(nums ...int) int {
    total := 0
    for i := 0; i < len(nums); i++ {
        total += nums[i]
    }
    return total
}

func main() {
    fmt.Println(sum(1, 2))         // 3
    fmt.Println(sum(5, 5, 5, 5))   // 20
    fmt.Println(sum())             // 0
}
```
**Вывод:**
```Run
3
20
0
```
📌 Такие параметры внутри функции выглядят как **срез** (`[]int`).

# Функции как значения

Функции в [[1.0. Go Basics|Go]] — «первоклассные граждане». Их можно присваивать [[1.1. Переменные|переменным]], передавать в другие функции и возвращать.
#### Пример присвоения:
```go
func double(x int) int { return x * 2 }

func main() {
    f := double
    fmt.Println(f(7))
}
```
**Вывод:**
```Run
14
```
#### Пример анонимной функции:
```go
func main() {
    mul := func(a, b int) int {
        return a * b
    }
    fmt.Println(mul(3, 4))
}
```
**Вывод:**
```Run
12
```

# Рекурсия: функция вызывает саму себя

**Рекурсия** — это когда функция обращается сама к себе.
Важно иметь **условие выхода**, **иначе** будет **бесконечный вызов**.
#### Пример (факториал):
```go
func fact(n int) int {
    if n <= 1 {
        return 1
    }
    return n * fact(n-1)
}

func main() {
    fmt.Println(fact(5))
}
```
**Вывод:**
```Run
120
```

# Ошибки

В [[1.0. Go Basics|Go]] принято возвращать ошибку как **дополнительное значение**.

**Пример:**
```go
func safeDivide(a, b int) (int, error) {
    if b == 0 {
        return 0, fmt.Errorf("деление на ноль")
    }
    return a / b, nil
}

func main() {
    res, err := safeDivide(10, 0)
    if err != nil {
        fmt.Println("Ошибка:", err)
        return
    }
    fmt.Println("Результат:", res)
}
```
**Вывод:**
```
Ошибка: деление на ноль
```
