# Что такое структура в [[1.0. Go Basics|Go]]

**Структура (struct)** — это пользовательский [[1.2. Типы данных|тип]], который объединяет несколько полей разных типов.

Каждое поле имеет **имя** и **тип**.
#### Пример:
```go
type Person struct {
    Name  string
    Age   int
    Email string
}
```
Здесь мы создали новый тип `Person`, у которого три поля: `Name`, `Age` и `Email`.

---
# Создание экземпляра структуры

Когда тип определён, мы можем **создать [[1.1. Переменные|переменную]]** этого типа.
#### Пример:
```go
p := Person{
    Name:  "Аня",
    Age:   25,
    Email: "anya@example.com",
}
fmt.Println(p)
```
**Вывод:**
```Run
{Аня 25 anya@example.com}
```
#### Пояснение построчно:
1. `p := Person{...}` — создаём переменную `p` типа `Person`.
2. В фигурных скобках указываем значения для каждого поля.
3. `fmt.Println(p)` — печатаем структуру.

---
# Доступ к полям структуры

Чтобы обратиться к конкретному полю, используем точку (`.`):
```go
fmt.Println(p.Name)
fmt.Println(p.Age)
```
**Вывод:**
```Run
Аня
25
```

---
# Изменение значений полей

Поля можно изменять напрямую:
```go
p.Age = 26
fmt.Println(p.Age)
```
**Вывод:**
```Run
26
```

---
# Инициализация другими способами

[[1.0. Go Basics|Go]] позволяет создавать структуры несколькими способами:
#### <u>1. Полное указание полей:</u>
```go
p := Person{Name: "Аня", Age: 25, Email: "anya@example.com"}
```
#### <u>2. Без указания имён полей (по порядку):</u>
```go
p := Person{"Аня", 25, "anya@example.com"}
```
	Лучше избегать этого способа, потому что при изменении порядка полей в структуре можно получить ошибки.
#### <u>3. Через `var` и последующее присвоение:</u>
```go
var p Person
p.Name = "Аня"
p.Age = 25
p.Email = "anya@example.com"
```

# Структуры внутри структур (вложенные структуры)

**Структуры могут содержать другие структуры:**
```go
type Address struct {
    City  string
    Street string
}

type Person struct {
    Name    string
    Age     int
    Address Address
}

p := Person{
    Name: "Аня",
    Age:  25,
    Address: Address{
        City:   "Москва",
        Street: "Ленина, 10",
    },
}
fmt.Println(p.Address.City)
```
**Вывод:**
```Run
Москва
```

# Встраивание структур (композиция)

Иногда удобно объединить несколько структур, чтобы одна **включала** в себя поля другой.
Такой приём в [[1.0. Go Basics|Go]] называется **встраиванием структур** (_composition_).

Он помогает не дублировать одинаковые поля и делает код чище.

**Пример:**
```go
type Contact struct {
    Email string
    Phone string
}

type Employee struct {
    Name string
    Age  int
    Contact  // встроенная структура
}

e := Employee{
    Name: "Анна",
    Age:  28,
    Contact: Contact{
        Email: "anna@company.com",
        Phone: "+7 900 123 45 67",
    },
}
fmt.Println(e.Email)
fmt.Println(e.Phone)
```
**Вывод:**
```Run
anna@company.com
+7 900 123 45 67
```
#### Как это работает:
- Мы **встроили** структуру `Contact` в `Employee`.
- Теперь поля `Email` и `Phone` доступны напрямую:
    можно писать `e.Email`, а не `e.Contact.Email`.
- Это просто способ **собрать объект из нескольких частей**, как будто мы вложили одну коробку в другую.

#### Зачем это нужно:
- Чтобы **не повторять одни и те же поля** в разных структурах.
- Чтобы **группировать связанные данные** (например, контактные данные).
- Чтобы **делить код на логические блоки**, делая его понятнее.

# Анонимные структуры

Иногда нужно быстро объединить несколько значений без создания отдельного типа.

Для этого есть **анонимные структуры**:
```go
book := struct {
    Title  string
    Price  int
}{
    Title: "Go для начинающих",
    Price: 1200,
}
fmt.Println(book.Title)
```

# Управление доступом к данным в структурах

Иногда нужно <u>защитить данные внутри структуры</u> — например, чтобы нельзя было случайно присвоить неправильное значение.

В Go это делают просто:
	мы **не даём прямой доступ** к полю и работаем с ним **через методы**.
#### Пример:
```go
type Account struct {
    balance float64
}

// Метод для чтения баланса
func (a Account) Balance() float64 {
    return a.balance
}

// Метод для изменения баланса
func (a *Account) Add(amount float64) {
    if amount > 0 {
        a.balance += amount
    }
}
```
#### Использование:
```go
acc := Account{}
acc.Add(1000)
fmt.Println(acc.Balance())
```
**Вывод:**
```Run
1000
```
#### Разбор:
- Поле `balance` хранится **внутри** структуры `Account`.
    Мы не изменяем его напрямую, а используем методы.
- Метод `Add()` получает **[[1.4. Указатели - работа с памятью и передача данных|указатель]]** на `Account` (`Account`), потому что структуры в Go **передаются по значению**. Без указателя изменения не затронут оригинальный объект.
- Метод `Balance()` просто **возвращает копию** значения, поэтому указатель ему не нужен.
#### Почему это удобно:
- Можно **контролировать правила** изменения данных (например, не разрешать отрицательные суммы).
- Код становится **безопаснее** — мы не можем случайно повредить внутренние значения.
- Если в будущем логика изменится (например, добавится комиссия), достаточно поменять только метод, а не весь код.

### Ещё один пример:
```go
type Counter struct {
    value int
}

func (c *Counter) Inc() {
    c.value++
}

func (c Counter) Value() int {
    return c.value
}

func main() {
    c := Counter{}
    c.Inc()
    c.Inc()
    fmt.Println(c.Value())
}
```
**Вывод:**
```Run
2
```

# Разница между методами и [[1.5. Функции и передача данных|функциями]]

| Функция                            | Метод                                       |
| ---------------------------------- | ------------------------------------------- |
| Не принадлежит типу                | Прикреплена к типу                          |
| `func printPerson(p Person)`       | `func (p Person) printPerson()`             |
| Вызывается как `printPerson(p)`    | Вызывается как `p.printPerson()`            |
| Можно использовать для любых типов | Обычно логически связана с конкретным типом |

---
#### Пример объединения: структура + методы:
```go
type Rectangle struct {
    Width  float64
    Height float64
}

func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}

func (r *Rectangle) Scale(factor float64) {
    r.Width *= factor
    r.Height *= factor
}

rect := Rectangle{Width: 5, Height: 3}
fmt.Println("Площадь:", rect.Area())

rect.Scale(2)
fmt.Println("Новая площадь:", rect.Area())

```
**Вывод:**
```
Площадь: 15
Новая площадь: 60
```
