# Debug
Инструмент [[2.0.0. JetBrains Goland|GoLand]] для пошагового выполнения программы с возможностью анализа её состояния во время работы.

В отличие от [[2.1.3. Run|Run]], режим **Debug** позволяет **приостанавливать выполнение программы**, **просматривать значения переменных**, **выполнять код по шагам** и находить **причины ошибок**.

---
# Основные возможности:
- Пошаговое выполнение программы.
- Установка точек останова (Breakpoints).
- Просмотр значений переменных.
- Просмотр стека вызовов (Call Stack).
- Выполнение выражений во время отладки.
- Изменение значений переменных "на лету".

# Breakpoint
**Breakpoint (точка останова)** — специальная отметка в коде, при достижении которой выполнение программы автоматически приостанавливается.

# Основные действия Debug

### 1. Continue
Продолжить выполнение программы до следующего Breakpoint или завершения.

### 2. Step Over
Выполнить текущую строку кода и перейти к следующей.
Если строка содержит вызов функции, она выполнится полностью без перехода внутрь.

### 3. Step Into
Перейти внутрь вызываемой функции и выполнять её пошагово.
Используется для анализа работы конкретной функции.

### 4. Step Out
Выполнить оставшуюся часть текущей функции и вернуться в место её вызова.
Полезно, когда дальнейшая отладка внутри функции не требуется.

### 5. Run to Cursor
Продолжить выполнение программы до строки, на которой находится курсор.
Позволяет быстро перейти к интересующему месту без установки Breakpoint.

# Просмотр переменных

Во время отладки [[2.0.0. JetBrains Goland|GoLand]] отображает:
- локальные переменные;
- параметры функций;
- поля структур;
- значения указателей;
- содержимое массивов, слайсов и map.

Все значения обновляются после каждого шага выполнения.

---
# Call Stack
Показывает цепочку вызовов функций, которая привела программу к текущему месту выполнения.

Позволяет понять:
- откуда была вызвана функция;
- последовательность выполнения;
- место возникновения ошибки.

---
# Evaluate Expression
Позволяет выполнить произвольное выражение во время остановки программы.

Например:
```go
len(users)
```
или
```go
user.Name
```

Без изменения исходного кода.

---
# Watches
Можно добавить выражения для постоянного наблюдения.

Например:
```go
len(slice)

counter

user.IsAdmin
```

Значения автоматически обновляются после каждого шага.

# Горячие клавиши Debug

|     Действие      |  macOS  |
| :---------------: | :-----: |
|       Debug       |  `⌃ D`  |
|     Continue      | `⌥ ⌘ R` |
|     Step Over     |  `F8`   |
|     Step Into     |  `F7`   |
|     Step Out      | `⇧ F8`  |
| Toggle Breakpoint | `⌘ F8`  |

# Когда используется Debug
Debug применяется для:
- поиска ошибок;
- анализа логики программы;
- проверки условий;
- просмотра изменения переменных;
- изучения работы нового кода;
- понимания выполнения сложных алгоритмов.
