# Problems
Окно [[2.0.0. JetBrains Goland|Golang]], в котором отображаются найденные IDE ошибки, предупреждения и другие проблемы в текущем проекте.

Оно помогает быстро находить и исправлять ошибки, не просматривая код вручную.

# Основные возможности
- Просмотр всех найденных ошибок.
- Просмотр предупреждений (Warnings).
- Быстрый переход к проблемному месту в коде.
- Фильтрация проблем по файлам.
- Обновление списка после изменения кода.
---
# Какие проблемы отображаются
- синтаксические ошибки;
- неиспользуемые переменные;
- неиспользуемые импортируемые пакеты;
- недостижимый код;
- отсутствующие зависимости;
- потенциальные ошибки в коде;
- ошибки, найденные анализатором Go.
---
# Типы сообщений

## 1. Error
Критическая ошибка, из-за которой код не компилируется или работает некорректно.

Например:
```go
fmt.Prinln("Hello")
```
`Prinln` не существует.

---
## 2. Warning
Предупреждение о потенциальной проблеме.

Например:
```go
x := 10
```
Если переменная `x` нигде не используется, GoLand сообщит об этом.

---
## 3. Weak Warning
Некритичное замечание, связанное со стилем кода или рекомендациями IDE.

Например:
- избыточный код;
- возможное упрощение конструкции;
- рекомендации по рефакторингу.
---
# Работа с окном Problems
Из окна можно:
- открыть файл с ошибкой;
- перейти к нужной строке;
- просмотреть описание проблемы;
- быстро исправить её с помощью **Quick Fix** (`⌥ Enter`), если доступно.
---
# Полезно знать
- Список проблем обновляется автоматически во время редактирования кода.
- После исправления ошибки она исчезает из окна Problems.
- Некоторые предупреждения не мешают компиляции программы, но помогают писать более качественный и читаемый код.
---
# Окно Problems применяется для:
- поиска ошибок в проекте;
- быстрого перехода к проблемным участкам кода;
- контроля качества кода;
- проверки проекта перед запуском или коммитом.
