# Git
Окно [[2.0.0. JetBrains Goland|GoLand]] для работы с системой контроля версий Git без использования терминала.

Позволяет отслеживать изменения, создавать коммиты, работать с ветками и синхронизировать локальный репозиторий с удалённым.

# Основные возможности
- Просмотр изменённых файлов.
- Создание коммитов.
- Push и Pull.
- Работа с ветками.
- Просмотр истории изменений.
- Сравнение версий файлов (Diff).
- Разрешение конфликтов слияния.

# Изменения (Local Changes)
В окне Git отображаются все изменения относительно последнего коммита.

Для каждого файла можно увидеть его статус:
- **Modified** — файл изменён.
- **Added** — новый файл.
- **Deleted** — файл удалён.
- **Renamed** — файл переименован.

Перед коммитом можно просмотреть все изменения.

---
# Commit
Сохраняет текущее состояние проекта в локальном репозитории Git.

Перед созданием коммита необходимо:
- выбрать изменённые файлы;
- написать сообщение (Commit Message);
- нажать **Commit** или **Commit and Push**.

---
# Push
Команда **Push** отправляет локальные коммиты в удалённый репозиторий (например, GitHub или GitLab).

До выполнения Push изменения существуют только на локальном компьютере.

---
# Pull
Команда **Pull** получает последние изменения из удалённого репозитория и объединяет их с локальной копией проекта.

---
# Branches
GoLand позволяет:
- создавать новые ветки;
- переключаться между ветками;
- объединять (Merge) ветки;
- удалять ненужные ветки.

Текущая ветка отображается в строке состояния IDE.

---
# History
Вкладка **History** показывает:
- список коммитов;
- автора коммита;
- дату и время;
- сообщение коммита;
- изменения, внесённые в каждом коммите.

---
# Diff
Функция **Diff** позволяет сравнить:
- текущую версию файла;
- предыдущую версию;
- любую выбранную ревизию.

Изменения отображаются построчно:
- зелёный — добавленные строки;
- синий — изменённые строки;
- красный — удалённые строки.

---
# Merge Conflicts
При возникновении конфликта GoLand открывает встроенный инструмент сравнения.

Можно:
- принять локальную версию;
- принять удалённую версию;
- объединить изменения вручную.

# Окно **Git** применяется для:
- сохранения изменений проекта;
- просмотра истории разработки;
- работы с ветками;
- отправки изменений в удалённый репозиторий;
- получения изменений от других разработчиков;
- анализа различий между версиями файлов.
